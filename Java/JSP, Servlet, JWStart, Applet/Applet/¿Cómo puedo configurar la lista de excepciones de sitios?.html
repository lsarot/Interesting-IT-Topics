¿Cómo puedo configurar la lista de excepciones de sitios?

Este artículo se aplica a:
  • Versiones de Java: 7.0, 8.0

La función Lista de excepciones de sitios se introdujo en Java 7 Update 51. Al agregar la URL de la aplicación a la lista de excepciones, los usuarios pueden ejecutar aplicaciones de Internet enriquecidas (RIA) que normalmente bloquean las comprobaciones de seguridad.

A continuación se detallan algunos casos en los que se permite ejecutar las aplicaciones agregando la URL de la aplicación a la lista de sitios:
  • Si la aplicación no está firmada con un certificado de una autoridad de certificación de confianza.
  • Si la aplicación está alojada localmente.
  • Archivo Jar que no tiene el atributo de manifiesto de permisos.
  • Aplicación firmada con un certificado caducado.
  • El certificado utilizado para firmar la aplicación no se puede seleccionar para la revocación.

Gestión de la lista de excepciones de sitios

La lista de excepciones de sitios se gestiona en el separador Seguridad del panel del control de Java. La lista aparece en el separador. Para agregar, editar o eliminar una URL en la lista, haga clic en Editar lista de sitios.

Agregar una URL

  1. Haga clic en el botón Editar lista de sitios.
  2. Haga clic en Agregar en la ventana Lista de excepción de sitios.
  3. Haga clic en el campo vacío del campo Ubicación para introducir la URL.
    La URL debe empezar por http:// o https://
    por ejemplo http://myexample.com o https://myexample.com
  4. Haga clic en Aceptar para guardar la URL que ha introducido. Si hace clic en Cancelar, las URL no se guardan.
  5. Haga clic en Continuar en el cuadro de diálogo Advertencia de seguridad.

Formato de URL
  • Se necesita un protocolo y un dominio.
    Los protocolos soportados son FILE, HTTP y HTTPS. Se recomienda utilizar HTTPS. Si el protocolo no es HTTPS, se muestra un mensaje de advertencia.
  • El número de puerto es obligatorio solo si no se utiliza el puerto por defecto.
  • La ruta de acceso es opcional.
  • Los comodines no están soportados.
    • Si solo se proporciona un dominio, se podrá ejecutar cualquier RIA desde dicho dominio. Un dominio puede tener varias entradas como, por ejemplo, https://www.example.com y http://www.example.com.
    • Si la ruta acaba con una barra (/), por ejemplo, https://www.example.com/apps/, se podrán ejecutar RIA en ese directorio y en cualquier subdirectorio. Si la ruta no termina con una barra, por ejemplo, http://www.example.com/test/applet.html, solo se podrá ejecutar en ese RIA específico.

Solo agregue un sitio a la lista de excepciones si confía en el sitio en su totalidad. Incluso si se especifica la ruta, agregar un sitio que puede contener otras rutas no de confianza podría suponer riesgos de seguridad. No se recomienda hacerlo.

Si se introduce un URL no válida, aparece un icono de error junto al elemento. Si no se corrige la URL antes de hacer clic en Aceptar, la URL no válida no se habrá guardado.

Editar una URL

  1. Haga doble clic en la URL que desee editar en la ventana Lista de excepciones de sitios.
  2. Realice los cambios en la URL.
  3. Haga clic en Aceptar para guardar los cambios. Si hace clic en Cancelar, los cambios no se guardan.

Eliminar una URL

  1. Haga clic en la URL que desee eliminar en la ventana Lista de excepciones de sitios.
  2. Haga clic en Eliminar.
  3. Haga clic en Aceptar para guardar el cambio. Si hace clic en Cancelar, las URL no se eliminan de la lista.
INFORMACIÓN RELACIONADA

Si la URL donde está el applet no es la misma que la de la página web desde la que se inicia el applet, deberá añadir la URL del applet y la de la página web.

Ejemplo con distintas URL para el applet y la página web

Para Yahoo games Checkers, debe introducir ambas URL (las URL pueden ser distintas en función del host que albergue el juego)

  1. http://www.games.yahoo.com
    URL del dominio de juegos de Yahoo
  2. http://yog36.games.sp2.yahoo.com
    URL en la que está albergado el juego y que aparecerá en el cuadro de diálogo de la aplicación bloqueada
INFORMACIÓN TÉCNICA ADICIONAL

Juego de reglas de despliegue

Si hay instalado algún juego de reglas de despliegue activo en el sistema, las reglas de despliegue tienen prioridad sobre la lista de excepciones de sitios. La lista de excepciones de sitios solo se tiene en cuenta si se aplica la regla por defecto. Consulte el apartado sobre el juego de reglas de despliegue si desea obtener más información sobre las reglas de despliegue.
